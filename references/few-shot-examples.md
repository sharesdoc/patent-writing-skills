# Few-Shot 法律话术转换实例

本文档提供工程师语言→专利法律语言的转换范例，在撰写交底书时作为风格参照。

---

## 一、算法改进类

### 范例 1：损失函数改进

**原始输入（工程师语言）：**
> 我改了 Python 里模型的 Loss 函数，把原本的 MSE 换成了加上 L1 正则的自定义损失函数，防止模型过拟合。

**专利化输出（法律语言）：**
> 在模型训练阶段，构建联合损失函数，所述联合损失函数包括均方误差项以及用于对模型权重进行稀疏化约束的正则化惩罚项；利用所述联合损失函数迭代更新所述深度学习网络的参数，以抑制所述模型的过拟合倾向。

### 范例 2：注意力机制改进

**原始输入：**
> 我们优化了 Transformer 的 self-attention，原来 O(n²) 太慢了，现在用了局部窗口+稀疏注意力，速度提升明显。

**专利化输出：**
> 一种序列数据处理方法，包括：接收输入序列数据；将所述输入序列数据划分为若干个局部窗口；在每个所述局部窗口内执行自注意力计算，得到局部上下文表示；同时，通过预设的稀疏路由策略，在非相邻窗口之间选择性地建立远距离依赖连接；将所述局部上下文表示与所述远距离依赖连接进行融合，输出最终的序列特征表示。通过上述方法，将自注意力计算的时间复杂度从 O(n²) 降低至 O(n log n)，显著降低了服务器在处理长序列数据时的计算资源消耗。

### 范例 3：归一化方法改进

**原始输入：**
> 我把 LayerNorm 和 BatchNorm 融合了一下，搞了一个混合归一化，在小批量训练时表现更好。

**专利化输出：**
> 在所述深度神经网络的第i层与第i+1层之间，设置混合归一化层，所述混合归一化层用于：接收第i层的输出特征张量；分别沿所述输出特征张量的通道维度与空间维度执行统计量计算，得到通道统计量μ_c、σ_c与空间统计量μ_s、σ_s；利用可学习的权重系数α对所述通道统计量与空间统计量进行加权融合，得到混合统计量；基于所述混合统计量对所述输出特征张量执行归一化处理，输出归一化特征张量至第i+1层。

## 二、模型架构改进类

### 范例 4：多模态融合改进

**原始输入：**
> 我们把文本和图像在模型中用一个交叉注意力给融合起来了，比直接拼接的效果好多了。

**专利化输出：**
> 接收待处理的图像模态数据与文本模态数据；利用图像编码器对所述图像模态数据进行特征提取，生成视觉特征矩阵V；利用文本编码器对所述文本模态数据进行特征提取，生成语义特征向量T；将所述视觉特征矩阵V作为查询矩阵（Query），将所述语义特征向量T作为键矩阵（Key）和值矩阵（Value），输入预设的跨模态注意力计算模块，得到融合特征张量；其中，所述跨模态注意力计算模块通过计算所述视觉特征矩阵V中每个图像块特征与所述语义特征向量T中每个词语特征的注意力权重分布，实现模态间的细粒度对齐，从而保留模态间的语义对应关系。

### 范例 5：模型压缩

**原始输入：**
> 我把 BERT 用知识蒸馏压到了原来 1/10 的大小，精度只掉了 0.5%。

**专利化输出：**
> 利用经预训练的教师模型对未标注语料进行推理，生成软标签；构建包含若干层Transformer编码器的学生模型，所述学生模型的参数量小于所述教师模型的参数量；在训练过程中，计算所述学生模型输出与所述硬标签之间的第一损失值，以及所述学生模型输出与所述软标签之间的第二损失值；利用所述第一损失值与第二损失值的加权和作为总损失函数，对所述学生模型进行参数更新；其中，通过引入所述软标签中的类别概率分布信息作为监督信号，使得所述学生模型在学习真实标签的同时，拟合所述教师模型的类别间相对关系，从而在参数量压缩至原模型的预设比例的情况下，保持与所述教师模型接近的推理精度。

## 三、系统/流程改进类

### 范例 6：推理加速

**原始输入：**
> 我们做了一个推理加速方案，把 prompt 里重复的部分给缓存了，不用每次都算一遍。

**专利化输出：**
> 接收输入文本序列；将所述输入文本序列与缓存中的历史键值对（KV Cache）进行前缀匹配，识别出所述输入文本序列中的公共前缀部分与新增后缀部分；对于所述公共前缀部分，直接从所述缓存中读取对应的键值对张量，跳过自注意力计算步骤；仅对所述新增后缀部分执行自注意力计算，生成新增键值对张量；将所述缓存的键值对张量与所述新增键值对张量进行拼接，输入后续的解码层进行推理。通过复用公共前缀对应的计算中间结果，避免了重复的自注意力计算，降低了推理延迟及计算资源消耗。

### 范例 7：数据处理流程

**原始输入：**
> 我们用了一个 pipeline，把数据清洗、特征工程、模型训练自动串起来了，还有个监控面板看效果。

**专利化输出：**
> 一种自动化数据处理与模型训练系统，包括：数据接入模块，用于从至少一个数据源采集原始数据；数据清洗模块，与所述数据接入模块的输出端连接，用于对所述原始数据执行缺失值填充、异常值剔除及格式归一化处理，得到清洗后数据；特征工程模块，与所述数据清洗模块的输出端连接，用于对所述清洗后数据执行特征提取与特征选择操作，生成训练特征集；模型训练调度模块，与所述特征工程模块的输出端连接，用于在检测到所述训练特征集更新后，自动触发模型训练任务；以及监控反馈模块，用于采集所述模型训练任务执行过程中的性能指标，并在所述性能指标超过预设阈值时生成告警信号。

## 四、AI 风格校准：去 AI 味

专利需要极度冰冷、严谨的客观描述。以下展示"AI腔"到"法律语言"的转换。

### 敏感词替换表

| AI腔 / 主观词汇 | 法律语言 / 客观表达 |
|----------------|-------------------|
| 非常聪明地 / 智能地 | 根据预设规则自动 |
| 惊人地实现了 | 取得了 / 获得了 |
| 具有非常重大的意义 | 具有有益效果 |
| 显著提高了用户体验 | 降低了用户操作的响应延迟 |
| 系统可以帮用户... | 所述处理器，用于响应于用户操作，执行... |
| 我们使用了 / 我们的系统 | 本申请实施例中 / 所述系统 |
| 令人惊叹的性能 | 性能提升了X%（相比于现有技术） |
| 自动化的 | 自动执行的 |
| 聪明的算法 | 预设的优化策略 |

### 风格转换示例

**输入（AI腔）：**
> 系统可以非常聪明、自动化地帮用户挑选最快的方法，具有非常重大的意义，令人惊叹地实现了性能飞跃。

**输出（法律语言）：**
> 所述中央处理器根据预设的延迟阈值，自动从待选策略库中匹配目标执行路径。相比于现有的固定路由方案，本申请实施例将端到端响应延迟平均降低了 35%。

### AI 典型用语检测清单

在 Stage 5 审查时检查文本中是否存在以下信号词：
- [ ] "delve into" / "深入探讨"
- [ ] "crucial" / "至关重要"
- [ ] "it is important to note" / "值得注意的是"
- [ ] "非常"、"特别"、"极其"
- [ ] "令人"（如令人惊叹、令人满意）
- [ ] "显著提高"（无具体数据支撑）
- [ ] "我们"、"我司"、"用户"（人称代词）
- [ ] 破折号过度使用（每页 >2 个）
