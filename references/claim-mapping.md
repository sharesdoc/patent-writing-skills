# 权利要求映射与扩展指南

本文档定义从技术方案到权利要求书的映射方法论，以及保护范围最大化策略。

---

## 一、权利要求金字塔结构

```
        独立权利要求（上位概念，保护范围最宽）
       /        |        \
    从属权利1   从属权利2   从属权利3  （中位概念，增加技术细节）
    /    \      /    \      /    \
  实施例1 实施例2 ...                  （下位概念，充分公开）
```

**核心原则：**
- 独立权利要求：用最宽泛的上位概念，不含具体参数
- 从属权利要求：逐步增加技术特征细节
- 实施例：写死具体参数和优选方案，作为退守底线

---

## 二、独立权利要求撰写模板

### 方法独立权利要求（权利要求1）

```
1. 一种[技术领域]方法，其特征在于，包括以下步骤：
步骤S1：[动词][对象]，得到[中间结果1]；
步骤S2：基于所述[中间结果1]，[动词][对象]，得到[中间结果2]；
...
步骤Sn：基于所述[中间结果n-1]，[动词][对象]，输出[最终结果]。
```

**撰写要点：**
- 步骤数量：5-7 步为佳
- 每步必须有明确的输入→处理→输出
- 步骤之间必须有数据依赖关系
- 使用"接收"、"获取"、"生成"、"确定"、"输出"等标准动词
- 禁止出现"优选"、"可以"等非确定性词汇（留给从属权利要求）
- 禁止出现具体数值（如"阈值为0.5"）
- 禁止出现具体算法名（如"使用Adam优化器"）

### 装置独立权利要求（通常为权利要求 N）

```
N. 一种[技术领域]装置，其特征在于，包括：
XX模块，用于[执行步骤S1的功能描述]；
YY模块，用于[执行步骤S2的功能描述]；
...
[模块名称]模块，用于[执行步骤Sn的功能描述]。
```

**撰写要点：**
- 模块数量与方法步骤数量保持一一对应
- 模块名称必须与方法步骤产生直接映射关系
- 模块之间的连接关系需在说明书中交代

### 设备权利要求（通常为权利要求 N+1）

```
N+1. 一种电子设备，包括处理器和存储器，所述存储器中存储有计算机程序，
其特征在于，所述处理器执行所述计算机程序时实现权利要求1至M任一项所述方法的步骤。
```

### 介质权利要求（通常为权利要求 N+2）

```
N+2. 一种计算机可读存储介质，其上存储有计算机程序，其特征在于，
该计算机程序被处理器执行时实现权利要求1至M任一项所述方法的步骤。
```

## 三、从属权利要求撰写策略

### 策略一：逐步缩小（级联从属）

```
权利要求1：一种方法，包括步骤S1-S5。

权利要求2（引用1）：根据权利要求1所述的方法，其特征在于，步骤S2中所述[操作]
通过[具体机制A]实现。

权利要求3（引用2）：根据权利要求2所述的方法，其特征在于，所述[具体机制A]包括
以下子步骤：S21...[子步骤展开]...

权利要求4（引用1或2）：根据权利要求1或2所述的方法，其特征在于，步骤S3中所述
[参数P]的取值范围为[X, Y]。
```

### 策略二：多分支从属（保护多种变体）

```
权利要求1：一种方法，包括步骤S1-S5。

权利要求2（引用1）：根据权利要求1所述的方法，其特征在于，步骤S2中所述[融合方式]
为[加权求和方法A]。

权利要求3（引用1）：根据权利要求1所述的方法，其特征在于，步骤S2中所述[融合方式]
为[注意力机制方法B]。

权利要求4（引用2或3）：根据权利要求2或3所述的方法，其特征在于，在所述[融合]之后，
还包括[后处理步骤C]。
```

### 策略三：参数限定（作为退守底线）

从属权利要求是放置具体参数的最佳位置：

```
权利要求5（引用1）：根据权利要求1所述的方法，其特征在于，所述[特征提取网络]
采用基于ResNet-50的骨干网络结构。

权利要求6（引用1）：根据权利要求1所述的方法，其特征在于，步骤S3中所述[稀疏度阈值δ]
的取值为0.1至0.3。

权利要求7（引用1）：根据权利要求1所述的方法，其特征在于，所述[联合损失函数]中的
平衡系数λ取值为0.01至0.1。
```

## 四、权利要求数量建议

| 类别 | 建议数量 | 说明 |
|------|---------|------|
| 独立方法权利要求 | 1项 | 最核心的方法流程 |
| 从属方法权利要求 | 5-7项 | 逐步细化技术特征 |
| 独立装置权利要求 | 1项 | 与方法一一对应的模块装置 |
| 设备权利要求 | 1项 | 电子设备（处理器+存储器） |
| 介质权利要求 | 1项 | 计算机可读存储介质 |
| **总计** | **10+项** | 满足最低数量要求 |

## 五、常见错误与修正

| 错误 | 问题 | 修正 |
|------|------|------|
| 独立权利要求含具体数值 | 保护范围过窄 | 将数值移至从属权利要求或实施例 |
| 步骤之间无数据依赖 | 步骤可以任意重排 → 保护范围不确定 | 确保"S2基于S1的结果..." |
| 装置模块与方法步骤不对应 | 装置与方法保护范围不一致 | 建立一一映射关系 |
| 从属权利要求引用链断裂 | 审查时会要求修正 | 确保每个从属权利要求的引用基础存在 |
| 使用"优选"在独立权利要求中 | 降低保护范围确定性 | "优选"仅用于从属权利要求和实施例 |
| 遗漏设备/介质权利要求 | 保护范围不完整 | 必须包含电子设备和存储介质权利要求 |

## 六、保护范围最大化 Checklist

- [ ] 独立方法权利要求（权利要求1）的步骤是否使用了最宽泛的上位概念？
- [ ] 独立装置权利要求（权利要求N）的模块是否与方法步骤一一对应？
- [ ] 是否有至少 5 个从属权利要求逐步增加技术特征？
- [ ] 是否有针对关键参数的具体取值范围限定在从属权利要求中？
- [ ] 是否有至少 2 个不同方向的从属权利要求分支（如覆盖不同算法变体）？
- [ ] 是否包含了电子设备和计算机可读存储介质权利要求？
- [ ] 独立权利要求中是否避免了所有具体数值、具体品牌/框架名？
- [ ] 从属权利要求的引用关系是否完整且无循环引用？
