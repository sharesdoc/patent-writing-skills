# 专利交底书质量终检清单

此清单在 Stage 5（合规审查与定稿）使用。全部勾选方可放行交付。

---

## 一、结构完整性检查

- [ ] 板块顺序：发明名称 → 技术领域 → 背景技术 → 发明内容 → 附图说明 → 具体实施方式 → 权利要求书
- [ ] 无非法板块：无附录、参考文献、写案注意事项、专利布局建议、致谢、README
- [ ] 具体实施方式有标准开头语："为了使本发明的目的、技术方案和优点更加清楚，下面将结合附图和具体实施例对本发明作进一步详细描述。"
- [ ] 具体实施方式有标准结尾语："以上所述仅为本发明的实施例而已，并不用于限制本发明……"
- [ ] 每个板块的篇幅比例符合标准（具体实施方式 55-60%，总篇幅 ~800 行）

---

## 二、内容一致性检查

- [ ] 发明内容步骤与具体实施方式步骤一一对应
- [ ] 每个"进一步地"在发明内容中映射到权利要求书中的一个从属权利要求
- [ ] 每个权利要求特征在具体实施方式中有详细描述
- [ ] 有益效果逐条对应背景技术中提出的痛点（三位一体闭环）
- [ ] 附图说明图号与文中引用（"参见图X"、"如附图X所示"）完全匹配
- [ ] 第二方面装置模块与方法步骤建立一一映射

---

## 三、公式与符号检查

- [ ] 每个公式后紧跟"式中"段落
- [ ] 每个符号定义了物理含义和单位
- [ ] 关键参数有建议取值及取值理由
- [ ] 公式前后各有一行空行
- [ ] 每个公式有 3-5 段周围解释文本（公式动机 + 物理含义 + 参数设定 + 工程注意事项）

---

## 四、图形规范检查

- [ ] 所有 Mermaid 图首行包含 `%%{init: {'theme': 'dark'...}}%%` 深色主题声明
- [ ] 包含至少 1 个系统架构图（graph TD）+ 至少 1 个核心算法时序图（sequenceDiagram）
- [ ] 架构图为静态拓扑，包含层级划分、模块唯一编号（101, 201, 301...）、连线数据标注
- [ ] 时序图开启动态编号（autonumber），核心创新步骤使用 `rect rgb(40, 40, 40)` 框选
- [ ] 每个 Mermaid 代码块下方紧跟"详细说明"文本段落
- [ ] 图文编号完全一致：图中模块301 = 文中"所述融合引擎301"

---

## 五、专利特有检查（防驳回红线）

### 5.1 反抽象驳回
- [ ] 无纯数学公式堆砌——每个公式前后有物理/技术场景描述
- [ ] 无纯商业规则——技术方案与物理/数字化技术载体深度绑定
- [ ] 无"智力活动规则"嫌疑——每个算法步骤有明确的技术输入输出

### 5.2 反公开不充分
- [ ] AI 模型不是"魔法黑盒"——交代了模型内部结构（如骨干网络类型、层数结构）
- [ ] 损失函数有明确数学形式和物理含义
- [ ] 输入数据维度、预处理方式有明确交代
- [ ] 本领域普通技术人员读完后能实现——给出了足够的技术细节

### 5.3 反保护范围过窄
- [ ] 独立权利要求无具体参数值（0.5, 224×224 等具体数字仅出现在从属权利要求或实施例）
- [ ] 无特定编程语言名称（Python/C++/Java 等）
- [ ] 无特定框架名称（PyTorch/TensorFlow/JAX 等）
- [ ] 无特定硬件型号（RTX 4090/A100/H100 等）
- [ ] 独立权利要求使用上位概念，具体实现写在实施例

---

## 六、法言法语规范检查

- [ ] **去人称化**：全篇无"我们"、"我司"、"用户"→ 统一为"本申请实施例"、"所述处理器"、"所述接收端"
- [ ] **前后指代一致**：图中叫"融合引擎301"，文本中无"融合模块"、"特征单元"等不一致称呼
- [ ] **去主观化**：无"非常智能"、"显著提高"（无数据支撑）、"令人惊叹"等主观形容词
- [ ] **动词标准化**：使用"接收"、"获取"、"生成"、"确定"、"输出"、"响应于"、"配置为"、"用于"
- [ ] **无 AI 腔**：无"delve into"、"crucial"、"it is important to note"、"值得注意的是"等 AI 典型用语

---

## 七、权利要求书专项检查

- [ ] 独立方法权利要求步骤 5-7 步，步骤间有数据依赖
- [ ] 从属方法权利要求 5-8 项，引用关系完整无断裂
- [ ] 独立装置权利要求模块与方法步骤一一对应
- [ ] 包含电子设备权利要求（处理器+存储器）
- [ ] 包含计算机可读存储介质权利要求
- [ ] 权利要求总数 ≥ 10 项
- [ ] 独立权利要求使用上位概念，无从属权利要求越位

---

## 八、背景技术检查

- [ ] 引用了具体专利号（如 CNXXXXXXXXXB）且至少有 4 件最相关现有技术
- [ ] 对每件现有技术有客观描述（2-3 句）+ 局限分析
- [ ] 检索结论明确声明了新颖性（含检索日期、检索范围、检索关键词）
- [ ] "背景技术痛点 → 技术方案特征 → 有益效果"三位一体闭环完整

---

## 九、篇幅与完整性检查

- [ ] 总篇幅约 800 行（±50 行）
- [ ] 具体实施方式占比 55-60%
- [ ] 至少包含 2-3 个实施例
- [ ] 至少包含 2 个 Mermaid 图（1 架构图 + 1 时序图）
- [ ] 所有公式有"式中"段落且每个符号有定义
