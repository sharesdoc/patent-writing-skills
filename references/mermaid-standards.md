# 软件与 AI 专利 Mermaid 绘图硬核标准

本文档定义专利交底书中所有 Mermaid 图形的**强制性视觉与结构规范**。

---

## 一、全局视觉声明

所有 Mermaid 图形代码块，**第一行必须强制写入**以下初始化声明：

```
%%{init: {'theme': 'dark', 'themeVariables': { 'background': '#1e1e1e', 'primaryColor': '#333333', 'textColor': '#ffffff', 'lineColor': '#ffffff' }}}%%
```

### 设计原则
- 深色背景（#1e1e1e）+ 白色文字/线条
- 严禁彩色、花哨样式
- **严禁代码截图**：不得以代码截图、IDE 截图、终端截图代替图形。必须使用 Mermaid 代码生成图
- 所有组件必须带有数字编号
- 专业、严肃、符合专利文档调性

---

## 二、系统架构图规范（graph TD）

### 约束规则
1. **必须横向或纵向划分层级/子系统**（如：客户端层、服务路由层、AI核心计算层、数据持久层）
2. **所有模块必须带有唯一数字编号**，推荐编号规则：
   - 100系：客户端/数据源层
   - 200系：服务路由/中间件层
   - 300系：AI核心计算层
   - 400系：数据持久/存储层
   - 500系：外部系统/第三方接口层
3. **连线上必须标注流转的数据类型或控制信号**（如 `|原始图像/文本|`、`|标准特征流|`）
4. **避免交叉线**——合理排列节点顺序

### 完整示例

```
%%{init: {'theme': 'dark', 'themeVariables': { 'background': '#1e1e1e', 'primaryColor': '#333333', 'textColor': '#ffffff', 'lineColor': '#ffffff' }}}%%
graph TD
    subgraph 客户端层 [100 客户端层]
        A[101 数据采集终端] -->|原始图像/文本| B[102 预处理模块]
        B -->|标准化特征| C[103 本地缓存队列]
    end

    subgraph 服务路由层 [200 服务路由层]
        C -->|API请求| D[201 API网关鉴权中心]
        D -->|分发请求| E[202 负载均衡器]
        E -->|路由决策| F[203 请求调度控制器]
    end

    subgraph AI核心计算层 [300 AI核心计算层]
        F -->|并行推理请求| G[301 多模态特征融合引擎]
        G -->|融合向量| H[302 深度学习推理大模型]
        H -->|多维注意力权重| I[303 损失函数约束校验器]
        I -->|校验通过信号| J[304 推理结果后处理器]
    end

    subgraph 数据持久层 [400 数据持久层]
        J -->|结构化推理结果| K[401 向量数据库]
        J -->|服务日志| L[402 关系型数据库]
        J -->|缓存热数据| M[403 分布式缓存节点]
    end
```

### 伴随文本说明规范

**在 Mermaid 代码块下方，必须出现如下格式的架构说明：**

> **系统架构详细说明：**
>
> - **所述客户端层100**，用于捕获用户的输入数据。其中，**数据采集终端101** 将原始图像输入至 **预处理模块102** 进行降噪与归一化处理；处理后的标准化特征暂存于 **本地缓存队列103** 中。
> - **所述AI核心计算层300**，是本发明的核心控制主体。**多模态特征融合引擎301** 的输出端与 **深度学习推理大模型302** 的输入端连接，用于将不同模态的特征进行跨模态注意力加权拼接。所述推理大模型302的输出端进一步连接至 **损失函数约束校验器303**，用于在推理过程中实时校验输出结果的置信度。
> - **所述数据持久层400**，包含 **向量数据库401** 与 **关系型数据库402**。向量数据库401用于存储高维特征向量以支持相似度检索；关系型数据库402用于持久化存储结构化推理结果及服务日志。

## 三、方法流程图规范（flowchart TD）

方法流程图用于展现算法或业务逻辑的**逐步骤执行流程**，必须有明确的开始和结束节点。区别于系统架构图（关注模块拓扑），方法流程图关注**步骤间的控制流和数据依赖**。

### 约束规则
1. **必须有明确的开始节点和结束节点**（如 `Start([开始])` 和 `Stop([结束])`）
2. **步骤节点必须带有步骤编号**（S1, S2, S3...），与权利要求和说明书中的步骤编号严格一致
3. **菱形节点用于决策/条件判断**，且每个分支路径上必须标注判断条件
4. **关键数据流入/流出标注在连线或节点上**
5. **核心创新步骤用 `style` 高亮**（在深色主题中使用更亮的填充色 `#444444`）

### 完整示例

```
%%{init: {'theme': 'dark', 'themeVariables': { 'background': '#1e1e1e', 'primaryColor': '#333333', 'textColor': '#ffffff', 'lineColor': '#ffffff' }}}%%
flowchart TD
    Start([S1: 开始]) --> S2[接收多模态输入数据]
    S2 --> S3[执行数据预处理:<br/>降噪与归一化]
    S3 --> S4{判断: 数据模态类型}
    
    S4 -->|文本模态| S5[通过文本编码器<br/>提取语义向量T]
    S4 -->|图像模态| S6[通过图像编码器<br/>提取视觉特征矩阵V]
    
    S5 --> S7[跨模态注意力融合]
    S6 --> S7
    
    S7 --> S8[输入深度学习推理模型]
    
    style S8 fill:#444444,stroke:#ffffff,color:#ffffff
    S8 --> S9[执行改进的自注意力计算:<br/>生成上下文关联矩阵M]
    
    style S9 fill:#444444,stroke:#ffffff,color:#ffffff
    S9 --> S10{稀疏度检查:<br/>矩阵M是否满足<br/>预设稀疏度阈值δ?}
    
    style S10 fill:#444444,stroke:#ffffff,color:#ffffff
    S10 -->|是| S11[输出推理结果]
    S10 -->|否| S12[执行稀疏度约束裁剪]
    S12 --> S9
    
    S11 --> S13[置信度校验与后处理]
    S13 --> Stop([S14: 输出最终结果, 结束])
```

### 伴随文本说明规范

**在 Mermaid 代码块下方，必须出现如下格式的流程说明：**

> **核心方法流程详细说明：**
>
> - **步骤S1-S3（数据预处理阶段）：** 所述数据采集终端101接收多模态输入数据后，由预处理模块102执行降噪与归一化处理，生成标准化特征数据。
> - **步骤S8-S10（核心创新步骤）：** 所述深度学习推理大模型302接收到融合后的高维特征张量后，执行改进的自注意力计算。具体地，在Transformer编码器的第K层隐藏层中，将全局注意力计算替换为基于稀疏度阈值δ的局部敏感哈希稀疏注意力（步骤S9）。随后由约束校验器303检查生成的上下文关联矩阵M是否满足预设稀疏度阈值δ（步骤S10）。若不满足，执行稀疏度约束裁剪（步骤S12）并返回步骤S9重新计算，形成迭代优化闭环。

## 四、核心流程与算法时序图规范（sequenceDiagram）

### 约束规则
1. **必须开启自动编号**：`autonumber`
2. **必须使用 `rect rgb(40, 40, 40)` 明确框选本发明的核心创新算法步骤**
3. **必须使用 `Note over` 标记每个步骤的简要说明**
4. **参与者必须使用与架构图一致的模块编号和名称**
5. **数据交互的箭头上必须标注数据类型或控制信号**
6. **区分同步调用（`->>`）和异步返回（`-->>`）**

### 完整示例

```
%%{init: {'theme': 'dark', 'themeVariables': { 'background': '#1e1e1e', 'actorBkg': '#333333', 'actorTextColor': '#ffffff', 'signalColor': '#ffffff', 'signalTextColor': '#ffffff' }}}%%
sequenceDiagram
    autonumber
    actor User as 用户终端 (101)
    participant Gateway as API网关 (201)
    participant Fusion as 融合引擎 (301)
    participant Model as 推理大模型 (302)
    participant Validator as 约束校验器 (303)
    participant DB as 向量数据库 (401)

    User->>Gateway: 发送多模态原始数据请求
    Gateway->>Gateway: 执行身份鉴权与请求解析
    Gateway->>Fusion: 转发标准化多模态数据
    
    Note over Fusion: 执行步骤 S201:<br/>对文本模态与图像模态<br/>进行时空对齐与特征级融合
    
    Fusion->>Model: 输入融合后的高维特征张量
    
    rect rgb(40, 40, 40)
        Note over Model: 执行步骤 S202 (核心创新点①):<br/>在Transformer编码器第K层中<br/>引入稀疏自注意力机制<br/>计算上下文关联矩阵M
        Model->>Model: 对矩阵M进行稀疏度约束裁剪
        Note over Model: 执行步骤 S203 (核心创新点②):<br/>调用联合损失函数L=L1+λL2<br/>校验预测结果与参考特征的一致性
    end
    
    Model->>DB: 检索匹配的最接近拓扑向量
    DB-->>Model: 返回Top-K邻近参考特征
    Model->>Validator: 输出初步推理结果
    Validator->>Validator: 执行置信度阈值校验
    Validator-->>Model: 返回校验通过/驳回信号
    Model-->>Fusion: 输出最终的目标识别结果
    Fusion-->>Gateway: 封装标准化响应报文
    Gateway-->>User: 返回结构化业务数据
```

### 时序图创新点标注规范

**使用 `rect rgb(40, 40, 40)` 框选核心创新步骤时：**
- 框内必须包含至少一个 `Note over` 标注
- `Note over` 中必须写明步骤编号（S201, S202...）和"核心创新点"标签
- 常规步骤（数据接收、转发、存储）不得框入 `rect` 区域

### 伴随文本说明规范

**在 Mermaid 代码块下方，必须出现如下格式的流程说明：**

> **核心流程与算法详细说明：**
>
> - **步骤1（S201）：** 所述API网关201接收到用户终端101发送的多模态原始数据请求，执行身份鉴权后，将标准化多模态数据转发至多模态特征融合引擎301。
> - **步骤3-5（S202-S203，核心创新点）：** 所述深度学习推理大模型302接收到所述融合后的高维特征张量后，在其Transformer编码器的第K层隐藏层中调用改进的自注意力机制——将传统的全局注意力计算替换为基于稀疏度阈值δ的局部敏感哈希稀疏注意力，从而将自注意力计算复杂度从O(n²)降低至O(n log n)。在计算上下文关联矩阵M后，对矩阵元素进行稀疏度约束裁剪，仅保留超过预设阈值的注意力权重。随后调用联合损失函数L=L₁+λL₂，其中L₁为任务损失项，L₂为稀疏度正则化惩罚项，λ为平衡系数，在推理阶段通过最小化联合损失函数来校验预测结果与参考特征的一致性。
> - **步骤7：** 所述向量数据库401基于余弦相似度检索并返回Top-K个最接近的拓扑向量作为参考特征，辅助模型进行上下文感知的归一化解码。

## 五、图号管理规范

### 图号分配规则
- 架构图：图1 为系统架构图
- 流程图：图2 为核心方法流程图
- 时序图：图3 为核心算法/交互时序图
- 附加图：图4+ 为实施例中的辅助示意图

### 附图说明格式

```
图1为本发明实施例中一种XXX系统的架构示意图；
图2为本发明实施例中XXX方法的核心流程示意图；
图3为本发明实施例中核心算法步骤的时序交互示意图；
图4为本发明实施例中另一个应用场景的系统部署示意图。
```

## 六、常见违规与修正

| 违规 | 修正 |
|------|------|
| 图中模块无编号 | 为每个模块添加唯一数字编号（101, 102...） |
| 架构图中出现业务实体名（如"淘宝订单"） | 替换为技术抽象（如"交易数据"、"业务请求"） |
| 时序图未使用 `autonumber` | 在 `sequenceDiagram` 下方添加 `autonumber` |
| 核心创新步骤未框选 | 使用 `rect rgb(40, 40, 40)` 包围核心步骤 |
| 连线无标注 | 在每个箭头后添加 `|数据类型|` 标注 |
| 使用彩色主题 | 强制使用 dark 主题初始化代码 |
| 图文编号不一致 | 检查图中编号与文中所引编号是否精准匹配 |
