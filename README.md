# patent-writing v2.0

> Claude Code Skill for Chinese Patent Technical Disclosure Writing（中国发明专利技术交底书撰写助手）

## 这是什么？

一个将 Claude 转变为专业中国发明专利交底书撰写专家的 Claude Code Skill。它提供：

- **五阶段管线** — 痛点挖掘 → 方案泛化 → 图文起草 → 实施例扩展 → 合规审查，阶段间设质量门
- **苏格拉底式追问** — 深度挖掘发明点，剥离业务外壳，逼迫发明人说出真正的技术创新
- **标准化结构** — 严格遵循 CNIPA 规范的六大板块格式
- **Mermaid 图形系统** — 自动生成深色主题系统架构图 + 核心算法时序图，模块带数字编号
- **软件/AI 防御机制** — 反专利法第25条驳回、反公开不充分、反保护范围过窄
- **Few-Shot 法律话术转换** — 将工程师语言转化为法言法语
- **模拟审查员审计** — Stage 5 自动对交底书进行合规扫描和风险分级
- **多模式命令** — 7 个子命令覆盖全流程/单阶段/答辩等场景

## 核心设计思想

本 Skill 的核心目标是**将工程师的"开发/实验思维"重塑为专利局认可的"技术/法律思维"**。

工程师写交底书最常犯的错误是把交底书写成"软件说明书"或"论文"。本 Skill 通过三个维度的思维跃迁来解决这个痼疾：

1. **靶向思维**：高价值专利的起点不是"我做了什么"，而是"老办法有什么致命缺陷"
2. **法律思维**：从"代码实现"到"技术泛化"——专利法不保护具体代码，只保护技术方案
3. **技术性防线**：算法必须与物理/数字化技术场景深度绑定，规避纯智力活动规则驳回

## 交底书标准六大板块

```
# 发明名称                    (~1%, ~5行)
## 技术领域                    (~1%, ~5行)
## 背景技术                    (~12-15%, ~100行)
## 发明内容                    (~15-18%, ~120行)
## 附图说明                    (~3%, ~25行)
## 具体实施方式                 (~55-60%, ~450行)  ← 核心板块
## 权利要求书                  (~8-10%, ~60行)
```

目标总篇幅：~800 行（针对数学密集型方法专利）

## 五阶段质量门

```
Stage 1: 痛点挖掘 → 质量门①：痛点是否客观/技术化？
Stage 2: 方案泛化 → 质量门②：是否消除业务外壳/上位化？
Stage 3: 图文起草 → 质量门③：图文是否齐全/编号一致？
Stage 4: 实施例扩展 → 质量门④：保护范围是否足够宽？
Stage 5: 合规审查 → 质量门⑤：全部终检清单通过？
```

红线警告触发后，必须打回修改，不得跳过进入下一阶段。

## 设计原则

- **每个公式需要 3-5 段周围解释文本**
- **每个"进一步地"映射到一个从属权利要求**
- **发明内容步骤与具体实施方式步骤一一对应**
- **有益效果必须映射到背景技术中提出的痛点**
- **独立权利要求使用宽泛上位语言——不包含具体参数值**
- **所有 Mermaid 图强制使用深色主题 `%%{init: {'theme': 'dark'}}%%`**
